# 🌌 Gemini-API: Image OCR & Translation Pipeline

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Google Cloud](https://img.shields.io/badge/GoogleCloud-%234285F4.svg?style=flat&logo=google-cloud&logoColor=white)](https://cloud.google.com/)

An automated data pipeline that processes signage images using Google Cloud's AI services. This tool extracts text via **Cloud Vision API**, translates it to Japanese via **Cloud Translation API**, and exports the structured data to **BigQuery**.

## 🛠️ Process Workflow
1. **Extraction:** Read image files (.jpg/.png) from a Google Cloud Storage bucket.
2. **OCR:** Send images to the **Vision API** to detect text and identify the source locale.
3. **Storage:** Save the raw extracted text as individual `.txt` files back to GCS.
4. **Translation:** If the source text is not Japanese, the **Translation API** converts it to Japanese.
5. **Analytics:** Insert the final record (Original Text, Locale, Translation, Filename) into a **BigQuery** table.

## 🚀 Setup & Requirements

### 1. IAM Roles
The Service Account (`ml-api-sa`) requires:
* `roles/storage.admin`
* `roles/bigquery.dataOwner`
* `roles/serviceusage.serviceUsageConsumer`

### 2. Environment Configuration
Ensure your service account key is exported:
```bash
export GOOGLE_APPLICATION_CREDENTIALS="$(pwd)/key.json"
```

### 3. Usage
Run the script by passing your Project ID and Bucket Name:
```bash
python3 analyze-images-v2.py [PROJECT_ID] [BUCKET_NAME]
```

## 📊 Verification
After execution, verify the data in BigQuery:
```sql
SELECT locale, COUNT(locale) as lcount 
FROM image_classification_dataset.image_text_detail 
GROUP BY locale 
ORDER BY lcount DESC
```
