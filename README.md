# 🌌 Gemini-API: Image OCR & Translation Pipeline

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Google Cloud](https://img.shields.io/badge/GoogleCloud-%234285F4.svg?style=flat&logo=google-cloud&logoColor=white)](https://cloud.google.com/)

An automated data pipeline that processes signage images. This project supports both **Google Cloud Platform (GCP)** enterprise integration and a practical **Local Mode** for offline or quota-limited environments. This tool extracts text, translates non-Japanese text to Japanese, and makes the structured data available.

## 🛠️ Execution Modes

### 1. Simulatiom Mode  
Perfect for running the app without a Google Cloud project or when out of quota.
* **OCR:** Powered by Pytesseract (Tesseract OCR engine).
* **Translation:** Handled by `deep-translator`.
* **Database:** Results are saved to a local `final_results.csv`.

**Setup:**
1. Install Tesseract OCR on your system:
   - **Ubuntu/Codespaces:** `sudo apt install tesseract-ocr`
   - **Mac:** `brew install tesseract`
2. Install Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Run the app:
   ```bash
   python3 app/local-analyze.py
   ```

### 2. Google Cloud Mode (Enterprise)
Uses Vision API, Translation API, and BigQuery for enterprise-scale processing.

**Setup:**
The Service Account (`ml-api-sa`) requires:
* `roles/storage.admin`
* `roles/bigquery.dataOwner`
* `roles/serviceusage.serviceUsageConsumer`

**Environment Configuration:**
Ensure your service account key is exported:
```bash
export GOOGLE_APPLICATION_CREDENTIALS="key.json"
```

**Usage:**
```bash
python3 app/analyze-images-v2.py [PROJECT_ID] [BUCKET_NAME]
```
